What is Git?

Git is a distributed version control system. It tracks your code history so you can experiment, revert, and collaborate safely.

Why we use Git

Git lets you save your work in snapshots called commits. You can see who changed what, and go back to a stable version any time.

  • Undo mistakes easily
  • Work in teams without conflicts
  • Keep a clean project history

Common Git commands

git init
git status
git add .
git commit -m "message"
git push origin main

These are enough to start your hackathon project.

Git + GitHub

Git runs on your local machine.
GitHub is the cloud where you push your Git repositories, so your code is backed up and shareable with the world.

What is GitHub Actions?

GitHub Actions is an automation engine inside GitHub. It can build, test, and deploy your project every time you push code.

Automation

Instead of manually deploying your site, you write a YAML workflow file. GitHub reads that file and runs all the steps for you.

YAML workflow

The file .github/workflows/deploy.yml describes:

  • When the workflow should run
  • What jobs and steps to execute
  • How to build and deploy your website
